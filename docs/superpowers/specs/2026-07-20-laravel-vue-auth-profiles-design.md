# Laravel + Vue (Inertia) — Auth & Profiles Foundation

**Date:** 2026-07-20
**Project:** `eat`
**Status:** Approved

## Goal

Scaffold the foundation of the `eat` application: a Laravel + Vue.js stack with
account registration, session-based authentication, password reset, and a user
profile settings page. This is the base layer that future features build on.

## Stack

| Concern        | Choice                                    |
|----------------|-------------------------------------------|
| Backend        | Laravel 11+ (PHP 8.4)                      |
| Frontend       | Vue 3 + Inertia.js (SPA feel, no API layer)|
| Auth scaffold  | Laravel Breeze (Vue/Inertia stack)        |
| Bundler        | Vite                                      |
| Styling        | Tailwind CSS (bundled with Breeze)        |
| Database (dev) | SQLite (single file, zero-config)         |
| Language       | Plain JavaScript (no TypeScript)          |
| SSR            | Off                                       |

## Architecture

Inertia.js bridges Laravel and Vue: Laravel keeps its routing, controllers, and
**session/cookie-based** authentication, while page rendering happens in Vue 3
components. There is no separate REST API and no auth tokens to manage — one
codebase, one deploy.

```
Browser
  │  (Inertia visits)
  ▼
Laravel routes → controllers → Inertia::render('Page', props)
  │
  ▼
resources/js/Pages/*.vue  (Vue 3 SPA)
```

## Features (in scope)

- **Register** — name, email, password
- **Login / Logout** — session-based
- **Forgot / Reset password** — reset link via email (uses the configured mailer;
  in dev, the `log` mailer writes the link to `storage/logs/laravel.log`)
- **Profile settings page** (`/profile`) with three sections:
  - Update name & email
  - Change password
  - Delete account

## Data model

Uses Laravel's default auth schema — no custom fields for now:

- `users` — id, name, email, email_verified_at (unused), password, remember_token, timestamps
- `password_reset_tokens`
- `sessions`

Profile fields (avatar, bio, dietary prefs) are intentionally deferred and can be
added later via a migration + form-field additions.

## Key files (generated by Breeze)

- `app/Http/Controllers/Auth/*` — auth controllers
- `app/Http/Controllers/ProfileController.php` — profile update/delete
- `routes/web.php`, `routes/auth.php`
- `resources/js/Pages/Auth/*.vue` — Login, Register, ForgotPassword, ResetPassword
- `resources/js/Pages/Profile/*.vue` — Edit + partial components
- `resources/js/Pages/Dashboard.vue` — post-login landing page
- `resources/js/Components/*.vue` — reusable form/UI components
- `database/database.sqlite`

## Setup sequence

1. `composer create-project laravel/laravel .` — scaffold into current dir
2. Configure `.env` for SQLite; create `database/database.sqlite`; `php artisan migrate`
3. `composer require laravel/breeze --dev`; `php artisan breeze:install vue`
   (no dark mode, no SSR, no TypeScript)
4. `npm install && npm run build`
5. `php artisan test`; then `php artisan serve` + `npm run dev` to smoke-test

## Verification

- `php artisan test` passes — Breeze ships feature tests for registration, login,
  password reset, and profile update.
- Manual smoke test: register a user → log in → edit profile name/email → change
  password → log out.

## Out of scope (YAGNI)

Email verification, social login (OAuth/Socialite), avatar upload, bio/custom
profile fields, dark mode, SSR, TypeScript. All straightforward to add later.
